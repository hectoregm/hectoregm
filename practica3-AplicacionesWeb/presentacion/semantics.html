semántica y API Form

Semantica

Un Elemento Semántico describe de forma "clara" su significado tanto al navegador como al desarrollador, por lo cual se dice que un elemento semántico es auto-descriptivo.
  • Ejemplos de elementos no-semánticos : <div> y <span>
  • Ejemplos de elementos semánticos : <form> , <table> y <img>

HTML5 proporciona etiquetas para manejo de Web 3.0 que permiten describir cual es el significado del contenido, por ejemplo su importancia, su finalidad y las relaciones que existen.
No presentan importancia en cuando a diseño/visualización; su importancia radica en su contenido y en la manera mediante la cual los buscadores procesan este contenido.

Estas etiquetas estan pensadas para que los buscadores puedan indexar e interpretar su contenido aunado a la busqueda de apariciones de palabras en el contenido del documento.

HTML5 permite incorporar a las páginas archivos RDF/OWL para describir relaciones entre los términos utilizados.

Elementos Semánticos

La siguiente lista menciona algunos de las principales etiquetas que definen elementos semánticos para cada parte de la página
  • <header>
  • <nav>
  • <mark>
  • <details>
  • <section>
  • <article>
  • <aside>
  • <figcaption>
  • <figure>
  • < footer>

descripción

Estructura del documento

<header>

El elemento header especifica el encabezado de un documento o sección.
Este elemento header debe utilizarse como un contenedor para el contenido introductorio del documento o sección, por lo que se puede tener varios elementos header en un solo documento.

<footer>

El elemento footer especifica un pie de página en un documento o sección.
Este debe contener información acerca de su elemento contenedor, por lo que suele contener el autor del documento, información de copyright, enlaces a términos de uso, información de contacto, etc. Se puede tener varios elementos footer en un solo documento.

<section>

Define una sección de un documento.
Una sección es una agrupación temática de contenido, por lo general acompañado de un encabezado

<nav>

El elemento nav define un conjunto de vínculos de navegación.
El elemento nav está destinado a grandes bloques de enlaces de navegación. Sin embargo, no todos los enlaces en un documento deben estar dentro de un elemento nav

<aside>

El elemento aside define un cierto contenido aparte de los contenidos/información que se colocan en la sección principal (como en una barra lateral).
Este contenido puede no estar relacionado con el contenido principal.

Estructura del cuerpo

<article>

El elemento article especifica el contenido independiente, autónomo.
Un artículo debe tener sentido por sí mismo y en principio debe ser posible distribuirlo de forma independiente del resto del documento.
Ejemplos en los que se puede utilizar un elemento article:

  • Mensaje Foro
  • Blog post
  • Noticias
  • Comentario

<mark>

Resalta parte de un texto considerado reelevante o importante. Esta reelevancia o importancia puede estar asociada a las acciones del usuario

<small>

Representa contenido al margen, denominado coloquialmente como letras pequeñas.
Ejemplos de este tipo de contenido:

  • descargos
  • restricciones legales
  • declaración de derechos

<cite>

Usado para mostrar una cita textual, en particular el título de un trabajo

<address>

Define información de contacto para un elemento o para el documento completo

<time>

Muestra una fecha y hora en formatos comprensibles por el usuario y el navegador. Es necesario definir un valor para el usuario (ubicado entre la etiqueta) y otro específicamente para los programas y navegadores (asignando un valor al atributo datetime ).

<figure> y <figcaption>

La etiqueta figure especiíca contenido autónomo, como ilustraciones, diagramas, fotos, listas de códigos, etc
Mientras que el contenido del elemento figure está relacionada con el contenido principal, su posición es independiente de este, por lo que al ser eliminado debe afectar el flujo o el sentido del contenido del documento.

La etiqueta figcaption define un título para un elemento figure. El elemento figcaption puede ser colocado al principio o al final del elemento figure

Consideraciones

Los elementos explicados anteriormente son todos elementos de bloque (excepto <figcaption> .
Para que estos elementos funcionen adecuadamente en los principales navegadores, es necesario establecer la propiedad css display: block (esto hace que los navegadores antiguos representen estos elementos correctamente): 
header, section, footer, 
aside, nav, article, figure { 
  display: block; 
}

Problema con Internet Explorer 8
y versiones anteriores

La versión de Internet explorer 8 y anteriores a esta no saben cómo representar (aplicar) propiedades CSS en elementos que no reconoce, por lo que no admiten estilos en <header> , <section> , <footer> , <aside> , <nav> , <article> , <figure> , entre otros nuevos elementos de HTML5.

Afortunadamente,existe una solución JavaScript llamada HTML5 Shiv .
Se puede descargar y en: http://code.google.com/p/html5shiv/
Después de la descarga, es necesario insertar el siguiente código en el elemento <head> : 

<!--[if lt IE 9]>
<script src="html5shiv.js"></script>
<![endif]-->

El código anterior es un comentario que sólo Internet Explorer lee, para las versiones anteriores a la versión 9. Debe ser colocado en el elemento head porque Internet Explorer necesita saber acerca de los elementos antes de que procesarlos.

ejemplo

En esta ocasión agregaremos al archivo ejemplo.html el siguiente código (esto dentro del tag <body> ) con el objetivo de darle estructura:

En <header>
(previamente agregamos el archivo html5shiv.js en la carpeta ejemplos/js ) : 

<link rel="stylesheet" href="ejemplos/css/main.css" />

<!--[if lt IE 9]>
    <script src="ejemplos/js/html5shiv.js"></script>
<![endif]-->

En <body> :

creamos el archivo main.css dentro de la carpeta ejemplos/css con el siguiente código (algunos detalles se explicarán en la sección Estilos):

    display: block;

	text-transform: uppercase;
}

#main-navbar ul{
	list-style:none;
	
}

	display: inline;
}

#main-navbar ul li > a, 
#main-navbar ul li > a:link {
	text-decoration:none;
}

API FORM y nuevos elementos

<form>

tag para crear un formulario,. Su estructura no ha cambiado, pero ahora se nos proveen nuevos atributos:

autocomplete Valor booleano que habilita la opción de autocompletado para los inputs pertenecientes al formulario.
Tiene dos valores ( on / off ) y por defecto el valor es on
Puede ser específicado en el tag form o directamente en cada uno de los inputs.
novalidate Valor booleano presencial que inhabilita la capacidad de validación de cada input asociado al formulario. Para específicar esta característica en cada input hacemos uso delatributo formnovalidate
Valor presencial/valor booleano presencial: Este tipo de valor solo se declara, no hay necesidad de especificar el valor que toma puesto que esta dado por defecto y su presencia es suficiente para activar la funcionalidad asociada

<datalist>

Define una lista de opciones o sugerencias para un input. El input requiere especificar como valor de su atributo list el valor del atributo id del datalist correspondiente

<progress>

Representa el progreso de una tarea

Progresbar a 65%: Ejemplo de progresbar

<meter>

Define una medida escalar dentro de un rango especificado (de forma total o parcial)

meter con valor 30 meter a 30


El valor mínimo es: 0, el valor máximo es: 100

<output>

representa el resultado de un cálculo. Generalmente el cálculo se realiza haciendo uso de los datos proporcionados en el formulario. Mediante Javascript y el valor del atributo for se hace referecia a la(s) fuente(s) de(los) dato(s) que participa(n) en el cálculo correspondiente y asimismo se realizan las modificaciones requeridas

El valor devuelto es : 5 .

<input>

tag para crear un elemento mediante el cual el usuario proporciona algún tipo de dato.
Se pueden definir/proporcionar los siguientes atributos (proveeemos los atributos básicos y algunos nuevos que son definidos en la especificación de HTML5):

type Determina el tipo de dato esperado por parte del usuario.
Mediante atributos adicionales se apoya/ayuda para limitar y/o controlar los datos proporcionados por el usuario.
name Define el identificador del input (el nombre usado como identificador debe ser único para el formulario asociado)
value Valor que el input toma por defecto en caso de que el usuario no proporcione datos
En algunos casos tambien es tomado como valor inicial (depende del tipo especificado)
placeholder Texto que provee una sugerencia corta para ayudar o guiar al usuario.
Generalmente se muestra dentro del input como marca de agua (con cierta opacidad)
(aplica para text, email, search, url, password, tel, ...)
required Valor booleano presencial que obliga al usuario a ingresar datos en el input (i.e el input en cuestion no puede quedar vacío)
autofocus Valor booleano presencial mediante el cual se permite establecer el foco hacia el elemento (i.e la página será enfocada hacia el elemento correspondiente) mientras el usuario este interactuando con este
multiple Valor booleano presencial que permite el ingreso de más de una entrada/dato
Los datos deber estar separados por comas
pattern Definimos una expresión regular mediante la cual establecemos las reglas de validación del contenido proporcionado por el usuario
title Definimos un título para el input
El título debe ser una descripción corta y precisa del contenido aceptado
form Asociamos el input con el formulario al cual pertenece; el valor que toma es el valor del atributo name definido para el formulario asociado.
Es necesario cuando declaramos el input fuera del tag form al cual pertenece, por lo que podemos insertar inputs en cualquier parte de la página
autocomplete Valor booleano presencial que habilita la opción de autocompletado del contenido proporcionado por el ususario (las opciones de autocompletado dependerán propiamente de cada navegador)
formnovalidate Valor booleano presencial que inhabilita la validación del input correspondiente
list Recibe el valor del atributo id de un elemento <datalist> .
El contenido del elemento <datalist> , será mostrado como opciones/sugerencia/ayuda para el contenido proporcionado por el usuario
min valor mínimo aceptado por el input.
(aplica para number, range, date, month, week, time, meter, progresbar)
max valor máximo aceptado por el input
(aplica para number, range, date, month, week, time, meter, progresbar)
step valor que define la cantidad de incremento o decremento de los datos
(aplica para number, range)

definiciones para type

text Recibe texto de forma general
email Recibe texto con formato de dirección email. El texto proporcionado por el usuario será validado como un email
search Recibe texto. El contenido proporcionado es manejado como un indicador por parte del navegador
speech Recibe voz/sonido. Habilita el micrófono del dispositivo para capturar voz/sonido, el contenido capturado es intrepretado y traducido a texto
Nota: actualmente este input solo esta disponible en navegadores compatibles con webkit
url Recibe texto con formato de URL. El texto proporcionado por el usuario será validado como una URL
password Recibe texto, el contenido proporcionado es bloqueado
submit Envia el contenido del formulario proporcionado por el usuario
tel Recibe un número telefonico
color provee una herramienta gráfica para seleccionar un color (paleta de colores). El color seleccionado es traducido a su representación hexadecimal
number Recibe datos numéricos.
range Permite al usuario seleccionar un valor a partir de una serie de valores o rango
NOTA: No existe un diseño estandar para este tipo de input pero normalmente es mostrado en pantalla como un elemento deslizable o un campo con flechas para seleccionar un valor
date Recibe contenido en formato de fecha
Algunos navegadores proporcionan un calendario que permite al usuario seleccionar un día (fecha) que será ingresado y asignado al input
week Similar a date, pero solo permite seleccionar semanas
El contenido que generalmente devuelve es el año y el numero de semana codificado
month Similar a date, pero solo permite seleccionar meses
El contenido que generalmente devuelve es el año y el numero de mes codificado
time Similar a date, devuleve la hora eleccionada
El contenido develto depende del navegador, algunos devuelven las horas con minutos y segundos mientras que otros solo devuelven horas con minutos codificados
datetime recibe fecha, hora y zona horaria
datetime_local recibe fecha y hora completa (no incluye zona horaria)
hidden
button
checkbox
file
image
radio
reset
otros tipos definidos desde versiones anteriores

ValidityState

En realidad este es un "Objeto" que provee algunos atributos, algunos de los cuales nos permiten una validación en tiempo real

Es usual en estos casos controlar el estado valid ,el cual es un atributo de ValidityState que devuelve el estado de un elemento considerando cada uno de los posibles estados de validación

Si cada condición (al realizar la validación) es válida (valga la redundancia) entonces el valor del atributo valid será TRUE

Adicionalmente existe ocho estados posibles que dependen de diferentes condiiones; estos deben ser llamados en el estado valid especificando elemento.valid.estado (los estados se explican a continuación)

valueMissing El valor de este estado es TRUE cuando el atributo required es declarado y el campo está vacío.
typeMismatch El valor de este estado es TRUE cuando la sintaxis de la entrada no corresponde con el tipo especificado.
patternMismatch El valor de este estado es TRUE cuando la entrada no corresponde con el patrón provisto por el atributo pattern.
tooLong El valor de este estado es TRUE cuando el atributo maxlength fue declarado y la entrada es más extensa que el valor especificado para este atributo.
rangeUnderflow El valor de este estado es TRUE cuando el atributo min fue declarado y la entrada es menor que el valor especificado para este atributo.
rangeOverflow El valor de este estado es TRUE cuando el atributo max fue declarado y la entrada es más grande que el valor especificado para este atributo.
stepMismatch El valor de este estado es TRUE cuando el atributo step fue declarado y su valor no corresponde con los valores de atributos como min , max y value .
customError El valor de este estado es TRUE cuando declaramos un error personalizado usando el método setCustomValidity() .
willValidate Atributo diseñado para detectar de manera dinámica aquellos elementos que no tienen por que ser validados
Su sintaxis es elemento.willValidate
