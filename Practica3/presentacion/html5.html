Evolucion de la Web

Web 1.0: Web Estática

En la década de los 90's se difundió y consolidó la primera versión disponible de la Web: Web 1.0.
Esta versión consistía en documentos cuyo contenido y formato era estático.

Características:
  • uso de framesets o marcos
  • Uso de extensiones HTML1
  • Uso del formato GIF para imágenes con resoluciones bajas
  • Involucran al usuario solo como observador
  • Dedicado principalmene a instituciones gubernamentales, educativas, empresas (que tienen infraestructura necesria)

Web 2.0: Web dinámica

En el 2003 aparece la Web 2.0 donde todas las aplicaciones y sitios tienen como característica principal al usuario como elemento principal de interacción.
Con la aparición de nuevos lenguajes de programación diseñados para desarrollo Web (como ASP3 o PHP4 ), se crean páginas dinámicas, permitiendo la actualización y modificaión frecuente del contenido.

Características:
  • Propósito general: desarrollo y difundir herramientas útiles para el usuario promedio o no especializado
  • Infraestructura basada en:
    • Proporcionar software que atiende y procesa solicitudes (Servidor)
    • Navegadores que se apoyen enlos estándares definidos
    • Diseño de protocolos para paso de mensajes
    • Desarrollo de aplicaciones destinadas a usuarios finales
    • Desarrollo basado en el contenido
  • Capacidad para almacenar información, proporcionar contenidos visibles y permitir a los usuarios acceder a ellos
  • Dirigido a todo público
Cada programa, aplicación o lenguaje debe ser lo suficientemente atractivo y sencillo como para que cada individuo aprenda a manejarlo de forma rápida y efectiva.

Web 3.0: Web Inteligente

¿Quién dijo que todo termina en la Web 2.0? Los especialistas –programadores, científicos, y sobre todo los fanáticos internautas– siguen hoy en día imaginando una Basados en la idea de constuir una Web inteligente mediante una red superior a la actual, complementando la capacidaddes de la Web 2.0 con nuevas y mejores propuestas e innovaciones para ofrecer al usuario, la propuesta de los especialistas de construir una Web semántica parece ser la respuesta.

Características:
  • Funcionamiento de agentes computarizados (programas) capaces de almacenar información, interpretarla y relacionarla de manera significativa
  • Incorporar componentes semánticos (aplicaciones) que realien procesamiento de información de forma automática
  • Desarrollo de nuevas técnicas y paradigmas para representar la información, su localización, su circulación, su integración y su recuperación.
Sobre esto ya existen algunos avances: ejemplos son XML y RDF.

HTML5

Dado que la versión anterior de HTML fue producida en 1999, la web ha cambiado mucho desde aquellos años.
HTML5 es la nueva especificación (estándar) para HTML. Es una colaboración entre el Consorcio W3C y WHATWG.

WHATWG trabajaba con una especificación para formularios web y aplicaciones, y la W3C estaba trabajando en la especificación para XHTML 2.0 . En 2006, decidieron cooperar y crear una nueva versión de HTML, estableciendo los siguientes ejes de trabajo:

  • Las nuevas características se deben basar en HTML , CSS , DOM y JavaScript
  • Reducir la necesidad de plugins externos
  • Un mejor manejo de errores
  • Mayor marcado (uso de etiquetas) de contenido para reemplazar el scripting actual
  • Debe ser independiente del dispositivo
  • Proceso de desarrollo debe ser visible para el público

sobre html5

Se ha especificado dos variantes de sintaxis para HTML5:

  • HTML5: HTML por "defecto" ( text/html ) conocida como HTML5
  • XHTML5: manejada como XML ( application/xhtml+xml ) conocida como sintaxis XHTML5

Es importante aclarar que no es simplemente una nueva versión del HTML, HTML5 es una conjunto de diversas especificaciones (tanto herramientas como estándares) en torno al desarrollo Web. Esta diseñado como un framework para apoyar la innovación y fomentar el potencial que ofrece la web

Así HTML5 es una nueva versión de diversas especificaciones, entre las que se encuentran:

  • una versión mejorada de HTML 4
  • una versión mejorada de XHTML 1
  • una versión mejorada de DOM 2

Con la nueva especificación, HTML 5 pretende proporcionar una plataforma para el desarrollo de aplicaciones Web más parecidas a las aplicaciones de escritorio, en donde la ejecución de estos no implique falta de recursos o facilidades para resolver las necesidades reales de los desarrolladores.

Para ello se están creando APIs que permitan trabajar con cualquiera de los elementos de la página y realizar acciones de forma nativa, con la intención que cualquier navegador siga los estándares, facilitanto el desarrollo y uso tanto por parte de los desarrolladores como de los usuarios, incrementando la compatibilidad con los navegadores y la posibilidad de un mejor acceso a sistemas/aplicaciones/páginas.

HTML5 es todavía un trabajo en progreso. Sin embargo, los principales navegadores son compatibles con muchos de los nuevos elementos de HTML5 y APIs.

Nuevas Características

Algunas de las nuevas características de HTML5:

  • Nuevos elementos de contenido específico, algunos basados en la idea de web semántica.
  • Soporte para almacenamiento local
  • Elementos para generación de gráficos
  • Nuevos controles de formulario, como el calendario, la fecha, la hora, email, URL, búsqueda
  • Elementos para reproducción de elementos multimedia
  • Elementos para acceso a componentes y características de dispositivos
  • Integración de técnicas y tecnologías para accelerar tiempos de espera
  • Soporte de una gran variedad de estilos y efectos

Estructura

HTML5 esta pensado para estructurar nuestras páginas de forma más simple. En la versión anterior necesitabamos crear una estructura básica commo la siguiente: 
<!DOCTYPE 
  HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN"
  "http://www.w3.org/TR/html4/loose.dtd">

<html xmlns="http://www.w3.org/1999/xhtml">

  <head>
  
    <meta http-equiv="Content-Type" 
         content="text/html; charset=utf-8" />
    <title>New Web Project</title>
    <meta name="description" content="" />
    <meta name="author" content="" />
	
  </head>
	
  <body>
	
     <h1>New Web Project Page</h1>
     
     <p>content here!</p>
		
  </body>
	
</html>

la estructura básica de un HTML mediante el nuevo estandar es la siguiente: 
<!DOCTYPE html>

<html lang="en">

  <head>
  
    <meta charset="utf-8" />
	<meta http-equiv="X-UA-Compatible" 
	     content="IE=edge,chrome=1" />
    <title>New Web Project</title>
	<meta name="description" content="" />
	<meta name="author" content="" />
	<meta name="viewport" 
	     content="width=device-width; initial-scale=1.0" />
	
  </head>

  <body>
		
	<h1>New Web Project Page</h1>
     
    <p>content here!</p>
		
  </body>
  
</html>

los cuales generan en el navegador algo del siguente estilo:

New Web Project Page

content here!

Nuevos Elementos

Elemento DOCTYPE
Etiqueta Descripcion
<!DOCTYPE html> Indica el tipo de documento creado, compatible con el correspondiente tag para la versión HTML 4.01 pero mas simplificado
Elemento canvas
Etiqueta Descripcion
<canvas> Utilizado para dibujar gráficos a través de secuencias de comandos (normalmente JavaScript)
Elementos Multimedia
Etiqueta Descripcion
<audio> Para definir audio o sonido
<video> Para definir vídeo o película
<source> Para definir dentro de este tag varios recursos de <video> y <audio>
<embed> Para proporcionar un contenedor para una aplicación externa o plug-in
<track> Para definir pistas para <video> y <audio>
Elementos para Formularios
Etiqueta Descripcion
<datalist> Especifica una lista de opciones predefinidas para los controles input
<keygen> Define un campo generador de claves (para formularios)
<output> Define el resultado de un cálculo
Elementos semánticos
Etiqueta Descripcion
<article> Define un artículo
<aside> Define el contenido considerado aparte del contenido de la página
<bdi> Aísla una parte de texto que puede ser formateada en una dirección distinta de otro texto fuera de ella
<command> Define un botón de comando que el usuario puede invocar
<details> Define detalles adicionales que el usuario pueda ver u ocultar
<dialog> Define un cuadro de diálogo o ventana
<summary> Define un título visible para un elemento <details>
<figure> Especifica contenido autónomo, como ilustraciones, diagramas, fotos, listas de códigos, etc
<figcaption> Define un título para un elemento <figure>
<footer> Define un pie de página de un documento o de la sección
<header> Define un encabezado de un documento o de la sección
<mark> Define texto marcado/resaltado
<meter> Define una medida escalar dentro de un rango especificado (de forma total o parcial)
<nav> Define enlaces de navegación
<progress> Representa el progreso de una tarea
<ruby> Define una anotación de ruby (para la tipografía de Asia Oriental)
<rt> Define una explicación/pronunciación de los caracteres (para la tipografía de Asia Oriental)
<rp> Define qué mostrar en los navegadores que no soportan anotaciones ruby
<section> Define una sección en un documento
<time> Define una fecha/hora
<wbr> Define una posible línea de ruptura
<output> Representa el resultado de un cálculo
